MFA

(Redemption Calls Only)

Shareholder calling:
  • Entity accounts (i.e Trusts, Corporations, Non-Profits) are excluded
  • Primerica Representatives called as the Representative are excluded Note: Please remember we cannot take trade instructions from an Office Manager and that Primerica Representatives cannot authorize anyone to speak or act on their or the shareholder’s behalf.
Confirm one of the following scenarios is true on the REDEMPTION requests and that we have a cell phone number of file, then follow the scripting below
  • The redemption amount is greater than $30,000
  • An address on the account changed within the past 3 months
  • Bank instructions were added/changed on an account within the past 3 months
"I WOULD BE HAPPY TO HELP YOU WITH YOUR REDEMPTION TODAY, BEFORE WE BEGIN, MAY I ASK FOR YOUR CELL PHONE NUMBER?"

Note – If mobile phone number is not on populated in CRM or the mobile phone is not accessible by the shareholder, Call Center defaults to standard authentication according to CCQ Guidebook.

(The Call Center should NEVER provide any phone number to the caller prior to it being verified)

Option 1: No Cell Phone Number Currently on File

  1. If we do not have it on file - We should update the account and then proceed with the redemption request without sending the MFA Code.
  2. No memorandum will need to be added to the account.

Option 2: The Caller Does Not Have a Cell Phone

If caller does not have a cell phone number we should then proceed with the redemption and must add the following memorandum to the account –

Callers Name(s) – Currently does not have a cell phone

See bottom of procedure on how to add a memorandum

Option 3: We Have the Cell Phone Number on File

If we currently have the cell phone number on file –

"TO INCREASE OUR LEVEL OF SRCURITY - I WOULD LIKE TO SEND YOU A ONE-TIME AUTHORIZATION CODE. DO YOU HAVE YOUR CELL PHONE WITH YOU AND ARE YOU ABLE TO RECEIVE TEXTS?"


Continue with the redemption request as we do today (see the CCQ Guidebook), please inform the caller that on future withdrawals an authorization code may be required before placing a redemption and they should have their cell phone available before calling into to the customer service line.

“Ok, I totally understand that this is a new request and I will go ahead and continue to process your transaction today.” “Please keep in mind that going forward you may be required to provide us with an authorization code before we can process your redemption request, so in the future when calling in to do a redemption request please have your cell phone available in case we need to send an authorization code.”

Call Center Agent should add a memorandum to the account -

Shareholder did not have cell phone and was provided with a soft warning.


“Great, This will only take a moment”

“I’m going to send you a text message with a six digit authentication code to the cell phone we have on file”

"Please let me know once you have authentication code.”

“Can you please provide me with the six digit code that you received?” ”Thank you, I have XXXXXX, is that correct?” ”Thank you, your authentication code has been confirmed.”

The CSR will add a memorandum to the account in CRM, and will copy and paste the authentication code from the MFA tool.

Authentication Code XXXXXX has been confirmed

Continue with the redemption request.

NOTE – CSR will resend authentication code up to 2 times

”Unfortunately, I could not confirm your authentication code. I would like to send you another authentication code.”

If a valid authentication code is not provided after 2 attempts the financial transaction will not be processed.

The CSR will report this information to their manager by emailing the Primerica Box. The Call Center manager will forward the details to the Client Service Officer for further review with the management company.
